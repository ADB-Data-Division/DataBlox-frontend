datablox_od.scaling#

Module contents#

Provides functions for adjusting GPS counts to improve representativeness.

datablox_od.scaling.adjust_gps_counts(model, entity, time, gps_count, verbose=False)#

Performs GPS count adjustment.

gps_count should be a raw GPS count. Preprocessing is automatically performed by this function based on how the training dataset was preprocessed (model actually contains metadata on the preprocessing of the training dataset).

The adjusted GPS count returned by this function should be taken as is. Other than maybe rounding it to the nearest integer, no postprocessing is needed.

Parameters:

  • model (datablox_od.shared._types._Model) – Object containing the trained model, the interpolation table for GPS counts between split points, and some metadata that are relevant for GPS count adjustment.

  • entity (str) – Entity (for example, administrative region) for which GPS count adjustment is to be performed. This entity should be present in the training dataset.

  • time (str) – Time period for which GPS count adjustment is to be performed. This time period should be present in the training dataset.

  • gps_count (int) – GPS count to be adjusted. This should be a raw GPS count, as preprocessing is automatically performed by this function based on how the training dataset was preprocessed.

  • verbose – Whether to print progress messages to the standard output.

Return type:

float

Returns:

Adjusted GPS count.

This adjusted count should be taken as is. Other than maybe rounding it to the nearest integer, no postprocessing is needed.

datablox_od.scaling.build_model(gps_count_column, random_state=42, model_type='xgboost', **kwargs)#

Initializes the model for GPS count adjustment.

For now, the only supported model is extreme gradient boosting (XGBoost). Under the hood, this function uses the scikit-learn wrapper interface for XGBoost, as provided by the xgboost package. As such, the user can configure the model as they would configure an xgboost.XGBRegressor() model (refer to the documentation here).

However, we advise against modifying the following XGBoost parameters, which we have already set by default:

  • monotone_constraints: The (predicted) adjusted counts should be monotonically increasing with respect to the GPS counts.

  • enable_categorical: The model should be able to handle categorical features since one of our features (the entity/location) is categorical.

Parameters:

  • gps_count_column (str) – Name of the column containing the GPS counts.

  • random_state (int) – Random number seed.

  • model_type (str) – Model type. For now, the only accepted value is "xgboost" (for extreme gradient boosting).

  • **kwargs –

    Additional arguments passed to the under-the-hood implementation of the model:

    • If model_type is "xgboost", then these would be passed to xgboost.XGBRegressor() (the scikit-learn wrapper interface for XGBoost, as provided by the xgboost package).

Return type:

xgboost.sklearn.XGBRegressor

Returns:

Model for GPS count adjustment.

datablox_od.scaling.preprocess_dataset_for_gps_count_adjustment(dataset, entity_column, time_column, ground_truth_count_column, gps_count_column, add_one_to_gps_counts=True, include_only_ground_truth_greater_than_gps_counts=True, log_normalize=True, add_zero_gps_count_zero_ground_truth=True, verbose=False)#

Performs data preprocessing prior to GPS count adjustment.

The preprocessing steps are performed in this order:

  • If add_one_to_gps_counts is True, 1 is added to the GPS counts. Set this parameter to True if it is reasonable to assume that GPS counts of 0 are not 0 per se but small, nonzero counts (which we assume to be 1). Note that 1 is added to all the GPS counts (regardless of whether or not they are equal to 0).

  • If include_only_ground_truth_greater_than_gps_counts is True, then only entries with ground-truth counts greater than their corresponding GPS counts are retained. Set this parameter to True if it is reasonable to assume that GPS counts greater than or equal to their corresponding ground-truth counts are outliers that should be dropped from the dataset (for instance, when training the count adjustment model).

  • If log_normalize is True, then log normalization is performed on all numerical columns (1 is added before taking the logarithm to account for the possibility that the argument is 0).

  • If add_zero_gps_count_zero_ground_truth_entries is True, then synthetic data with both GPS and ground-truth counts equal to 0 are added to the dataset. Set this parameter to True if it is reasonable to assume that GPS counts of 0 also correspond to ground-truth counts of 0. This assumption can help calibrate the count adjustment model when the adjusted GPS counts overshoot the ground-truth counts.

Parameters:

  • dataset (pandas.core.frame.DataFrame) – Panel dataset containing the GPS counts (to be adjusted) and the ground-truth counts.

  • entity_column (str) – Name of the column containing the entities (for example, the administrative regions).

  • time_column (str) – Name of the column containing the time periods.

  • ground_truth_count_column (str) – Name of the column containing the ground-truth counts.

  • gps_count_column (str) – Name of the column containing the GPS counts.

  • add_one_to_gps_counts (bool) – Whether to add 1 to the GPS counts.

  • include_only_ground_truth_greater_than_gps_counts (bool) – Whether to include only entries with ground-truth counts greater than their corresponding GPS counts.

  • log_normalize (bool) – Whether to perform log normalization on all numerical columns. If this parameter is set to True, then 1 is first added before taking the logarithm to account for the possibility that the argument is 0.

  • add_zero_gps_count_zero_ground_truth_entries – Whether to add synthetic data with both GPS and ground-truth counts equal to 0.

  • verbose (bool) – Whether to print progress messages to the standard output.

Return type:

datablox_od.shared._types._Dataset

Returns:

Object containing the preprocessed dataset for GPS count adjustment and some metadata that are relevant for training and GPS count adjustment.

datablox_od.scaling.train_model(model, dataset, curve='linear')#

Trains the model for GPS count adjustment.

Parameters:

  • model (xgboost.sklearn.XGBRegressor) – Model for GPS count adjustment. We recommend instantiating this model using datablox_od.scaling.build_model().

  • dataset (datablox_od.shared._types._Dataset) – Panel dataset containing the GPS counts (to be adjusted) and the ground-truth counts. Before passing the panel dataset as an argument to this function, we recommend preprocessing it using datablox_od.scaling.preprocess_dataset_for_gps_count_adjustment().

  • entity_column – Name of the column containing the entities (for example, the administrative regions).

  • time_column – Name of the column containing the time periods.

  • ground_truth_count_column – Name of the column containing the ground-truth counts.

  • gps_count_column – Name of the column containing the GPS counts.

  • curve (str) – Curve used to interpolate for GPS counts that fall between the model’s split points. For now, the only accepted value is "linear" (which fits a linear curve for interpolation). Fitting a (monotonically increasing) curve is necessary since tree-based models like XGBoost do not inherently interpolate for values between split points.

Return type:

datablox_od.shared._types._Model

Returns:

Object containing the trained model, the interpolation table for GPS counts between split points, and some metadata that are relevant for GPS count adjustment.
