datablox_od.trajectory#

Module contents#

Provides functions for processing trajectory data.

datablox_od.trajectory.detect_staypoints(pings, coordinates_column, timestamp_column, max_distance_between_pings_in_km=1, min_length_of_stay_in_minutes=30, max_days_between_two_consecutive_pings=1, representative_point='median', verbose=False)#

Detects staypoints following a modified version of the algorithm proposed by Li et al. [1].

Important: This function assumes that the pings come from only a single device.

A staypoint is a location where a person stayed for some period of time (for example, one’s place of residence, workplace, or visited tourist attraction). In the context of this function, a staypoint is defined as a list of pings that satisfies all of the following criteria:

  • The pings should be spatially close, that is, they are within max_distance_between_pings_in_km.

  • The length of stay (that is, the difference between the timestamps of the first and last pings) should be at least min_length_of_stay_in_minutes.

  • Every pair of consecutive pings should be at most max_days_between_two_consecutive_pings apart.

The first two criteria are from the reference algorithm by Li et al. [1]. We add the third criterion in order to parameterize the assumption that the device stayed in the same location when no pings were recorded for a possibly extended period of time.

The algorithm for identifying the staypoints given the trajectory of a device is as follows:

  • Order the pings by timestamp, and iterate through them sequentially, greedily identifying maximal-length staypoints.

  • Assign a representative point for each staypoint (computed based on the representative_point parameter). representative_point can be set to mean, as in the reference algorithm. By default, it is set to median for improved robustness to noise [2].

Note that, in calculating the length of stay and the representative point, we modified the reference algorithm to exclude the first point outside the distance threshold.

Parameters:

  • pings (geopandas.geodataframe.GeoDataFrame) –

    Contains the pings from the device of interest. The pings should come from only a single device. At the minimum, this data frame should have columns for the:

    • Coordinates of the pings. The coordinates of each ping should be stored as a shapely.Point object.

    • Timestamps of the pings. The timestamp of each ping should be stored as a numpy.datetime64 object, which is the return type when pandas.to_datetime() is used.

  • coordinates_column (str) – Name of the column containing the coordinates.

  • timestamp_column (str) – Name of the column containing the timestamps.

  • max_distance_between_pings_in_km (float) – Maximum distance (in km) between the points within a staypoint.

  • min_length_of_stay_in_minutes (float) – Minimum length of stay (in minutes) within a staypoint.

  • max_days_between_two_consecutive_pings (float) – Maximum number of days between every pair of (temporally) consecutive pings within a staypoint.

  • representative_point (str) – How the representative point of a staypoint is computed. The accepted values are median and mean.

  • verbose (bool) – Whether to print progress messages to the standard output.

Return type:

geopandas.geodataframe.GeoDataFrame

Returns:

Staypoints.
