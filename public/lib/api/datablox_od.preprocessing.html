datablox_od.preprocessing#

Module contents#

Provides functions for preprocessing GPS data.

datablox_od.preprocessing.filter_pings_based_on_timestamp(pings, timestamp_column, start_time, end_time)#

Filters pings to include only those from the specified start time to before the specified end time.

Parameters:

  • pings (geopandas.geodataframe.GeoDataFrame | pandas.core.frame.DataFrame) – Contains the pings. At the minimum, this data frame should have a column for the timestamps of the pings.

  • timestamp_column (str) – Name of the column containing the timestamps.

  • start_time (str) – Only pings from start_time to before end_time will be retained. Pings at exactly start_nighttime will be retained. The time should be given in 24-hour HH:MM format (for example, 7:00 p.m. should be encoded as 19:00).

  • end_time (str) – Only pings from start_time to before end_time will be retained. Pings at exactly end_nighttime will not be retained. The time should be given in 24-hour HH:MM format (for example, 7:00 a.m. should be encoded as 07:00, not 7:00).

Return type:

pandas.core.frame.DataFrame

Returns:

Pings from the specified start time to before the specified end time.

datablox_od.preprocessing.get_nearby_places(coordinates, places, distance_threshold, projected_crs, predicate='intersects')#

Identifies nearby places.

Formally, X is considered near Y if X is located within distance_threshold of Y. Whether the entirety of X or only a part of it should be within distance_threshold is determined by the predicate parameter. If predicate is "intersects", then a part of X is sufficient; if predicate is "contains", then the entirety of X should be within distance_threshold.

Parameters:

  • coordinates (geopandas.geodataframe.GeoDataFrame) – For each row in cordinates, nearby entries among places will be identified. The coordinates should be stored as a shapely.Point object.

  • places (geopandas.geodataframe.GeoDataFrame) – For each row in cordinates, nearby entries among places will be identified. The coordinates should be stored as a shapely.Point object.

  • distance_threshold (float) – Maximum distance for a place to be considered nearby. The unit of measure of distance_threshold is dependent on projected_crs.

  • projected_crs (pyproj.crs.crs.CRS) – Projected coordinate system.

  • predicate (str) – If predicate is "intersects", then it suffices for only a part of the place to be within distance_threshold. If predicate is "contains", then the entirety of the place should be within distance_threshold.

Return type:

geopandas.geodataframe.GeoDataFrame

Returns:

Nearby places.

datablox_od.preprocessing.map_pings_to_areas(pings, cell_to_area_mapping, spatial_index_column, device_id_column, coordinates_column, timestamp_column, adjust_timezone=0, drop_duplicate_pings=True, spatial_indexing_system='h3', spatial_index_resolution=8, verbose=False)#

Maps pings to the areas where they were recorded. A ping has three attributes: (1) the identifier of the device that issued it, (2) its coordinates, and (3) its timestamp.

Parameters:

  • pings (geopandas.geodataframe.GeoDataFrame) –

    Contains the pings. At the minimum, this data frame should have columns for the:

    • Identifiers of the devices that issued the pings.

    • Coordinates of the pings. The coordinates of each ping should be stored as a shapely.Point object.

    • Timestamps of the pings. The timestamp of each ping should be stored as a numpy.datetime64 object, which is the return type when pandas.to_datetime() is used.

  • cell_to_area_mapping (pandas.core.frame.DataFrame) – Maps the cells (from the spatial index) to the areas. To prevent confusion, pings and cell_to_area_mapping should not have any common column names.

  • spatial_index_column (str) – Name of the column in cell_to_area_mapping that stores the identifiers of the cells (from the spatial index).

  • device_id_column (str) – Name of the column containing the device identifiers.

  • coordinates_column (str) – Name of the column containing the coordinates.

  • timestamp_column (str) – Name of the column containing the timestamps.

  • adjust_timezone (float) – Number of hours by which the timestamp is to be adjusted.

  • drop_duplicate_pings (bool) – Whether to drop pings with the same device identifier and timestamp.

  • spatial_indexing_system (str) – Spatial indexing system. For now, the only accepted value is "h3" (for Uber’s hexagonal hierarchical geospatial indexing system). This should match the spatial indexing system used to generate cell_to_area_mapping.

  • spatial_index_resolution (int) – Spatial index resolution at which spatial indexing will be performed. For H3, refer to this documentation. This should match the spatial resolution used to generate cell_to_area_mapping.

  • verbose (bool) – Whether to print progress messages to the standard output.

Return type:

geopandas.geodataframe.GeoDataFrame

Returns:

Mapping between pings and the areas where they were recorded. This mapping is many-to-one, that is, multiple pings can map to the same area. Pings that do not map to any of the areas are dropped.

datablox_od.preprocessing.perform_geospatial_indexing(bounding_area, areas, spatial_index_column, spatial_indexing_system='h3', spatial_index_resolution=8, visualize=False, visualization_column=None, visualization_width=8, visualization_height=8, verbose=False)#

Performs geospatial indexing. In particular, this function maps cells (from the spatial index) to the given areas of interest.

Parameters:

  • bounding_area (Any class that implements __geo_interface__, as discussed here) – Entire area to be indexed (that is, it should cover all the areas of interest in areas). For example, if the areas of interest are subnational areas, then this should specify the national boundaries. The coordinates should be in EPSG:4326.

  • areas (geopandas.geodataframe.GeoDataFrame) – Contains the boundaries of the areas of interest. areas with boundaries outside bounding_area will not be indexed.

  • spatial_index_column (str) – Name of the column that will store the identifiers of the cells (from the spatial index). To prevent confusion, no column in areas should have this name.

  • spatial_indexing_system (str) –

    Spatial indexing system. For now, the only accepted value is "h3" (for Uber’s hexagonal hierarchical geospatial indexing system).

  • spatial_index_resolution (int) –

    Spatial index resolution at which spatial indexing will be performed. For H3, refer to this documentation.

  • visualize (bool) – Whether to plot the grid corresponding to the spatial index. Setting this to True can be helpful in verifying whether the cells indeed cover all the areas of interest or whether the specified resolution is sufficiently fine-grained.

  • visualization_column (str | None) – Name of the column containing the categories to be considered when plotting the grid. The colors in the plot correspond to the unique values in this column. Ignored if visualize is False.

  • visualization_width (int) – Width of the plot. Ignored if visualize is False.

  • visualization_height (int) – Height of the plot. Ignored if visualize is False.

  • verbose (bool) – Whether to print progress messages to the standard output.

Return type:

pandas.core.frame.DataFrame

Returns:

Mapping between cells (from the spatial index) and the areas of interest. This mapping is many-to-one, that is, multiple cells can map to the same area of interest.
