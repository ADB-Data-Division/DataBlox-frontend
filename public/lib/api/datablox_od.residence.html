datablox_od.residence#

Module contents#

Provides functions for inferring residence information.

datablox_od.residence.infer_residence(pings, device_id_column, timestamp_column, area_column, year, month, start_time='19:00', end_time='07:00', num_months_in_observation_window=6, min_num_days_present=30, min_fraction_pings_in_residence=0.5, include_statistics=False, temp_dir=None, skip_writing_temp_file_if_exists=True, delete_temp_dir=False, n_jobs=1, verbose=False)#

Infers residence information for the month of interest.

The algorithm for establishing the residence of a given device is as follows:

  • Consider the pings from num_months_in_observation_window months before and including the month of interest. For example, if the month of interest is June 2019 and num_months_in_observation_window is 6, then pings in the six-month observation window January 2019 to June 2019 will be considered.

  • We define the candidate residence as the area where most of the nighttime pings were recorded. By default, nighttime starts at 7:00 p.m. and ends before 7:00 a.m., following the guidelines of the United Nations Statistics Division [1], but this definition can be overridden with the start_time and end_time parameters.

  • For the candidate residence to be considered the residence, it needs to meet two additional criteria. First, the device should have issued nighttime pings for at least min_num_days_present days in the observation window; these pings do not necessarily have to be from the candidate residence. The default value of 30 days follows Lai et al. [2]. Second, at least min_fraction_pings_in_residence of the nighttime pings in the observation window should be from the candidate residence.

Parameters:

  • pings (geopandas.geodataframe.GeoDataFrame | pandas.core.frame.DataFrame) –

    Contains the pings mapped to the areas where they were recorded. At the minimum, this data frame should have columns for the:

    • Identifiers of the devices that issued the pings.

    • Timestamps of the pings. The timestamp of each ping should be stored as a numpy.datetime64 object, which is the return type when pandas.to_datetime() is used.

    • Areas where the pings were recorded. This function assumes that the pings have already been mapped to these areas, for example, via datablox_od.preprocessing.map_pings_to_areas(). The coordinates themselves are not needed.

  • device_id_column (str) – Name of the column containing the device identifiers.

  • timestamp_column (str) – Name of the column containing the timestamps.

  • area_column (str) – Name of the column containing the areas where the pings were recorded.

  • year (int) – Residence will be inferred for month of year. The full (four-digit) year should be given.

  • month (int) – Residence will be inferred for month of year. A value of 1 corresponds to January; 2, to February; and so on.

  • start_time (str) – Only pings from start_time to before end_time will be considered. Pings at exactly start_time will be considered. The time should be given in 24-hour HH:MM format (for example, 7:00 p.m. should be encoded as 19:00).

  • end_time (str) – Only pings from start_time to before end_time will be considered. Pings at exactly end_time will not be considered. The time should be given in 24-hour HH:MM format (for example, 7:00 a.m. should be encoded as 07:00, not 7:00).

  • num_months_in_observation_window (int) – Number of months in the observation window, that is, the number of months before and including the month of interest that will be considered for establishing device residence.

  • min_num_days_present (int) – For a given device’s candidate residence to be considered the residence, a condition is that the device should have issued pings for at least min_num_days_present in the observation window; these pings do not necessarily have to be from the candidate residence.

  • min_fraction_pings_in_residence (float) – For a given device’s candidate residence to be considered the residence, another condition is that at least min_fraction_pings_in_residence of the pings in the observation window should be from the candidate residence.

  • include_statistics (bool) – Whether to include statistics in the data frame being returned. These statistics include the number of days each device issued pings and the number of pings from the inferred residence for every month in the observation window. Note that setting this to True can significantly increase the runtime and memory consumption.

  • temp_dir (str | None) –

    Directory where temporary files will be saved. A value of None means that the temporary files will be saved in the current working directory. Specifically, this function creates a directory named DataBlox-OD-temp inside the specified temp_dir. It, in turn, contains three subdirectories:

    • daily-device-frequency – The Parquet files inside contain information on the number of pings from each area. This is computed for all devices at a daily granularity.

    • monthly-device-frequency – The Parquet files inside contain information on the number of pings from each area. This is computed for all devices at a monthly granularity.

    • monthly-device-presence – The Parquet files inside contain information on the number of days each device issued at least one ping. This is computed at a monthly granularity.

    Additionally, each Parquet file is coupled with an empty file with extension .done, the presence of which indicates that the Parquet file was generated without any issues.

  • skip_writing_temp_file_if_exists (bool) – Whether to skip writing a temporary file if it has already been generated (under the hood, this means that the corresponding .done file exists). Setting this to True allows the function to essentially resume should it be interrupted midway.

  • delete_temp_dir (bool) – Whether to delete the temporary directory.

  • n_jobs (int) – Controls the degree of parallelization when running this function. This value is equal to the number of jobs to be run in parallel. A value of -1 means that all processors will be used.

  • verbose (bool) – Whether to print progress messages to the standard output.

Return type:

pandas.core.frame.DataFrame

Returns:

Inferred residence of each device. Devices for which residence cannot be inferred are dropped.

If include_statistics is set to True, then additional columns are displayed, corresponding to the months in the observation window:

  • <year>-<month>_num_days_present – Number of days the device issued at least one ping. The pings included in this count do not necessarily have to be from the residence of the device.

  • total_num_days_present – Total number of days the device issued at least one ping in the observation window. The pings included in this count do not necessarily have to be from the residence of the device.

  • <year>-<month>_num_pings_in_residence - Number of pings from the residence of the device.

  • fraction_of_pings_in_residence - Fraction of pings in the observation window that were recorded from the residence of the device.
